## Learned Workspace Facts
- Projet Laravel sous XAMPP : URL publique `http://localhost/agrofinanceplus/public` ; API `http://localhost/agrofinanceplus/public/api` (`base_url` sans slash final). Commandes PHP en CLI : `C:\xampp1\php\php.exe` ; sous PowerShell enchainer avec `;` plutot que `&&` ; pour `php artisan tinker --execute`, echapper les `$` (ex. `` `$u ``) ou session tinker interactive. Inspection du schema SQL : phpMyAdmin (`http://localhost/phpmyadmin`) lorsque la CLI MySQL n'est pas utilisee.
- Si MySQL/MariaDB renvoie "Unknown collation utf8mb4_0900_ai_ci", aligner `charset` et `collation` dans `config/database.php` sur une collation supportee par le serveur (ex. utf8mb4_unicode_ci).
- Authentification API Sanctum : routes sous `api/auth` ; les handlers d'exceptions JSON pour l'API doivent reconnaitre les URLs dont le chemin contient le segment `api` (ex. `.../public/api/...` sous XAMPP). Pour `user_id` et filtres SQL par proprietaire, utiliser `auth()->user()->id` ; ne pas utiliser `Auth::id()`, car l'identifiant d'auth est le telephone.
- Auth web : les redirections "invite a se connecter" doivent cibler la route nommee `connexion` (pas `login`). Parcours inscription : `/inscription` puis `/verification-otp` puis `/creer-pin` puis connexion ; en environnement local, le code OTP est aussi journalise dans `storage/logs/laravel.log` (lignes `[OTP LOCAL]`).
- Typographie UI (app desktop, auth, dashboard) : Space Grotesk pour titres, logo et grands chiffres ; Inter pour navigation, formulaires et corps ; definies dans `resources/css/app.css` et les layouts.
- Le dashboard et la consolidation par exploitation (`activitesActives` / `calculerExploitation`) n'incluent que les activites au statut `en_cours`; les activites `termine` sont hors consolidation. Pour creer des ressources via l'API, utiliser POST avec corps JSON ; un GET avec corps ne cree pas la ressource et peut renvoyer une liste vide.
- Rapports PDF : stockage sous `storage/app/rapports/` (disque local), servis via les controleurs ; pas d'exposition directe via `public` ni besoin de `storage:link` pour ces fichiers. Lien de partage public (sans auth) : `GET {APP_URL}/partage/{lien_token}` (token issu de la generation, pas un placeholder Postman).
- Abonnement : logique (statut actif, plan, droits PDF, quotas d'exploitations) dans `App\Services\AbonnementService`. En base, la colonne `plan` est alignee sur des valeurs canoniques (`gratuit`, `essentielle`, `pro`, `cooperative`) ; le service applique les regles metier (PDF, multi-exploitations, historique limite). FedaPay en developpement sans cles : `FEDAPAY_MOCK=true`, `POST /api/abonnement/initier` et `POST /api/abonnement/finaliser-mock` ; en production `FEDAPAY_MOCK=false`, cles FedaPay, puis `php artisan config:clear`. Le callback `GET /api/abonnement/callback` est hors middleware Sanctum ; `POST /api/abonnement/initier` reste protege.
- Depots Git : ignorer `*.sql` (dumps locaux) et `.cursor/` ; ne jamais committer `.env`.
- Sidebar desktop : `toggleSidebar` / `window.toggleSidebar`, largeurs 260px (ouvert) et 72px (reduit), `mainContent` avec `margin-left` aligne, persistance `localStorage` cle `sidebarReduced` ; implemente dans le script inline de `resources/views/layouts/app-desktop.blade.php` ; `resources/js/app.js` ne contient que le token API (meta).
- Fonds d'ecran desktop : `App\Support\WeeklyBackgroundImages` lit `public/images/` (Agro / elevage / transformation), selectionne 4 URLs par semaine ISO (Agro, Agro, elevage, transformation) ; carrousel JS dans `app-desktop` pour tous les types d'exploitation. Pages auth (inscription, connexion, etc.) : image Unsplash fixe dans `app-auth.blade.php` (photo champ/ciel), ne pas la remplacer par la rotation hebdo.
