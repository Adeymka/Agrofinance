## Learned User Preferences
- Toujours repondre en francais.
- Pour le centre d'aide (articles publics), rediger en francais simple pour exploitants ; eviter les encarts techniques (logs locaux, chemins serveur) et les descriptions de visuels par type d'exploitation si ce n'est pas aligne sur l'application — privilegier les categories de transactions et le parcours utilisateur. Pour les FAQ et textes marketing publics (contact, accueil, a-propos), ne pas utiliser FSA / FSA-UAC ; preferer « indicateurs financiers agricoles ».
- Preferer un accompagnement pas a pas pour les tests et validations.
- Fournir les instructions de test une par une plutot qu'un bloc complet.
- En cas de 401 Non authentifie sur l'API, rappeler la verification du header Authorization (Bearer + token), de la variable d'environnement Postman, et la reconnexion via POST /auth/connexion si le token a ete revoque.
- Pour la recette des sprints API, proposer des etapes numerotees (ex. Sprint 5 : tests 1 a 8) avec resultats attendus par code HTTP.
- Le design mobile ne doit pas etre un simple responsive du web : chaque ecran doit etre repense pour les conventions mobiles (card lists, wizard forms, bottom nav) avec le meme langage visuel dark glassmorphisme que le desktop.

## Learned Workspace Facts
- Projet Laravel sous XAMPP : URL publique `http://localhost/agrofinanceplus/public` ; API `http://localhost/agrofinanceplus/public/api` (`base_url` sans slash final). Commandes PHP en CLI : `C:\xampp1\php\php.exe` ; sous PowerShell enchainer avec `;` plutot que `&&` ; pour `php artisan tinker --execute`, echapper les `$` (ex. `` `$u ``) ou session tinker interactive ; avec des backslashes de namespaces (`\App\...`), envelopper l'expression dans des guillemets simples PowerShell. DANGER : l'operateur `-replace` PowerShell dans une chaine double-quote interpole `$variable` comme vide — utiliser un script PHP temporaire pour les remplacements de fichiers en masse. Assets PWA (manifest, sw.js) : utiliser `{{ asset('fichier') }}` et non `/fichier` car l'app est dans un sous-dossier XAMPP ; `start_url` et `scope` du `manifest.json` = `/agrofinanceplus/public/`. Inspection du schema SQL : phpMyAdmin (`http://localhost/phpmyadmin`).
- Si MySQL/MariaDB renvoie "Unknown collation utf8mb4_0900_ai_ci", aligner `charset` et `collation` dans `config/database.php` sur une collation supportee par le serveur (ex. utf8mb4_unicode_ci).
- Authentification API Sanctum : routes sous `api/auth` ; les handlers d'exceptions JSON pour l'API doivent reconnaitre les URLs dont le chemin contient le segment `api` (ex. `.../public/api/...` sous XAMPP). Pour `user_id` et filtres SQL par proprietaire, utiliser `auth()->user()->id` ; ne pas utiliser `Auth::id()`, car l'identifiant d'auth est le telephone.
- Auth web : les redirections "invite a se connecter" doivent cibler la route nommee `connexion` (pas `login`). Parcours inscription : `/inscription` puis `/verification-otp` puis `/creer-pin` puis connexion ; en environnement local, le code OTP est aussi journalise dans `storage/logs/laravel.log` (lignes `[OTP LOCAL]`).
- Typographie UI : Space Grotesk pour titres, logo et grands chiffres ; Inter pour navigation, formulaires et corps ; definies dans `resources/css/app.css`. Layouts : `app-desktop` (sidebar glass, fond photo carrousel), `app-mobile` (dark glassmorphisme, bottom dock, header glass), `app-public` (pages marketing). Detection plateforme : middleware `App\Http\Middleware\DetectPlatform` enregistre via `$middleware->web(append:...)` dans `bootstrap/app.php` ; partage `$layout` ('layouts.app-mobile' | 'layouts.app-desktop') et `$platform` ('mobile' | 'desktop') a toutes les vues ; forcer via `?platform=mobile`. Toutes les vues app utilisent `@extends($layout)`. Pour dual-render dans une vue : `@if($platform === 'mobile') ... @else ... @endif`.
- Le dashboard et la consolidation par exploitation (`activitesActives` / `calculerExploitation`) n'incluent que les activites au statut `en_cours` ; les activites `termine` sont hors consolidation. Pour creer des ressources via l'API, utiliser POST avec corps JSON ; un GET avec corps ne cree pas la ressource et peut renvoyer une liste vide.
- Rapports PDF : stockage sous `storage/app/rapports/` (disque local), servis via les controleurs ; pas d'exposition directe via `public` ni besoin de `storage:link` pour ces fichiers. Lien de partage public (sans auth) : `GET {APP_URL}/partage/{lien_token}` (token issu de la generation, pas un placeholder Postman).
- Abonnement : logique (statut actif, plan, droits PDF, quotas d'exploitations) dans `App\Services\AbonnementService`. En base, la colonne `plan` est alignee sur des valeurs canoniques (`gratuit`, `essentielle`, `pro`, `cooperative`) ; le service applique les regles metier (PDF, multi-exploitations, historique limite). Tarifs affiches et factures (FCFA / mois) : Gratuit 0 ; Essentielle 1 500 ; Pro 5 000 ; Cooperative 8 000. FedaPay en developpement sans cles : `FEDAPAY_MOCK=true` ; en production `FEDAPAY_MOCK=false`. Le callback `GET /api/abonnement/callback` est hors middleware Sanctum.
- Depots Git : ignorer `*.sql` (dumps locaux) et `.cursor/` ; ne jamais committer `.env`.
- Layout desktop : sidebar glass (260px ouvert / 72px reduit), `toggleSidebar` / `window.toggleSidebar`, persistance `localStorage` cle `sidebarReduced`, implemente dans `app-desktop.blade.php`. Fonds d'ecran : `App\Support\WeeklyBackgroundImages` lit `public/images/`, selectionne 4 URLs par semaine ISO ; carrousel JS dans `app-desktop`. Pages auth : image Unsplash fixe dans `app-auth.blade.php`, ne pas remplacer par la rotation hebdo.
- Centre d'aide : textes HTML dans `database/seeders/Support/HelpArticleBody.php` ; commandes `php artisan help:seed-premiers-pas` et `help:seed-campagnes` pour pousser le contenu en base. Pages marketing (`accueil`, `a-propos`, `contact`) : layout `app-public.blade.php` ; CSS par page avec `@push('styles')` et `@stack('styles')` dans le `<head>` apres Vite.
- PWA : `public/manifest.json` (start_url `/agrofinanceplus/public/`, icones 72-512px, shortcuts Saisie/Activites) et `public/sw.js` (Cache First pour assets Vite/polices, Network First pour pages HTML, Network Only pour `/api/*`, Stale-While-Revalidate pour images, fallback `/offline`). Route `/offline` definie dans `routes/web.php` → vue `public/offline.blade.php`. Aucune route `alertes.index` n'existe dans `routes/web.php` (la vue `alertes/index.blade.php` existe mais n'est pas routee) ; utiliser la route `abonnement` comme fallback pour la cloche de notification mobile.
